**Purpose:** Response scripts for common objections

**Common Objections & Responses:**

**Objection 1: "How do I know the money actually goes to kids?"**
```
RESPONSE:
"Great question. Unlike traditional charities where you have to trust their word, we use smart contracts on the Base blockchain.

Every transaction is publicly verifiable at [BaseScan link].

You can see:
- Exact amount allocated to charity (60%)
- Timestamp of allocation
- Destination wallet address
- Cumulative impact (children helped)

Zero trust required. Just code."
```

**Objection 2: "Why isn't it 100% to charity if you care so much?"**
```
RESPONSE:
"Because we're not a charity - we're a sustainable tech business.

30% goes to infrastructure (servers, APIs, development) so the platform stays online.
10% goes to the founder so he can work full-time on this mission.

If we gave 100%, we'd shut down in 3 months when the money ran out.

60% to kids, sustainably, forever > 100% to kids for 3 months then nothing."
```

**Objection 3: "Is this a revenue allocation?"**
```
RESPONSE:
"No, and we never claim it is. You're not making a contribution - you're purchasing a product/service.

We're a for-profit LLC, not a 501(c)(3). We allocate OUR profits to Verified Pediatric Charities.

Think of it like this:
- Buy from Amazon ? 0% to charity
- Buy from us ? 60% to charity

You get the same service, kids get helped."
```

**Objection 4: "This sounds too good to be true."**
```
RESPONSE:
"I understand the skepticism. That's WHY we built this on blockchain.

Don't trust us. Trust the code.

Smart Contract: [BaseScan link]
GitHub Repository: [link] (fully open source)
Real-time Dashboard: [link] (see every transaction)

We've made it impossible to lie. Check for yourself."
```

**Objection 5: "Why blockchain? Seems like unnecessary complexity."**
```
RESPONSE:
"Fair question. Here's why:

Without blockchain:
- You have to trust our word
- We could change the split secretly
- No public audit trail
- 'Trust us, we're honest'

With blockchain:
- Backend system enforces 60/30/10 split
- Immutable - we can't change it even if we wanted to
- Public proof for anyone to verify
- Zero trust required

It's not about hype. It's about Transparent verification."
```

**Objection 6: "How is this different from [Competitor]?"**
```
RESPONSE:
"[Competitor] contributes X% after they calculate expenses and take profit.

We allocate 60% BEFORE anything else. It's hardcoded in the smart contract.

Also:
- We're fully transparent (blockchain proof)
- They show you a yearly report - we show you real-time transactions
- Our split is immutable - theirs can change anytime

We don't ask you to trust us. We prove it with code."
```
